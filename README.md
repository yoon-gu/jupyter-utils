# jupyter-utils

폐쇄망 + 타 조직 제공 Jupyter 환경이라는 **제약 위에서 최대한의 효율**을 내기 위한
Jupyter Lab + 터미널 유틸리티 모음입니다. (매직 커맨드·함수·**셸 스크립트**·확장·스니펫 등)
노트북 작업은 매직/함수로, 터미널 작업은 셸 스크립트로 — **둘 다 1급 형태**로 다룹니다.

## 환경 제약 (Context)

이 저장소의 모든 결과물은 아래 제약을 전제로 합니다. 이슈/구현 시 항상 염두에 둬주세요.

- 🚫 **폐쇄망 + 의존성 보안검토**: `pip install`로 바로 받지는 못합니다. **prebuilt wheel 반입은 가능**하지만, 들여오는 **모든 의존 패키지(전이 의존 포함)가 정보보안 검토를 통과**해야 합니다 → 의존성 최소화가 핵심. 표준 라이브러리 우선.
- 🔒 **권한 제약**: 관리자/sudo 권한이 없습니다. 설정은 보통 홈 디렉터리(`~/.ipython`, `~/.jupyter`) 한정입니다.
- 🔁 **서버 재시작**: Jupyter 서버는 타 조직이 띄워줍니다. 다만 확장은 `pip install` 후 **브라우저 새로고침으로 로드됨을 확인**(labextension 도입 가능). 서버 측 변경은 여전히 제약일 수 있습니다.
- 👥 **공유 환경**: 여러 사람이 함께 쓰므로 전역 변경은 지양합니다.

설치 부담이 적은 순: **스니펫 → 셸 스크립트 → 매직 커맨드 → 함수/모듈 → (가능하면) 확장**.
용도로 고르되, 가능하면 **하나의 코어를 여러 폼팩터(매직·함수·셸/CLI)가 공유**하도록 만듭니다.

## 배포 / 반입 (Offline delivery)

폐쇄망 반입은 두 가지 경로를 함께 지원합니다.

1. **startup 스크립트 떨구기** — 가장 가벼운 경로. `~/.ipython/profile_default/startup/`에 `.py`를 두면 설치 없이 자동 로드.
2. **prebuilt wheel + 오프라인 번들 반입** — ⭐ 권장. 이 repo를 설치형 패키지로 빌드하고, 의존성까지 **하나의 번들로 모아 반입**합니다.
   - ⚠️ **진짜 게이트는 의존 패키지의 정보보안 검토**입니다(wheel 반입 자체는 가능). → **의존성 최소화**가 최우선, 전이 의존이 적은 순수 파이썬 패키지 선호, 때로는 소규모 코드 vendoring이 패키지 통째 반입보다 검토가 쉽습니다.
   - 외부에서 `python -m build` 로 `jupyter_utils` wheel을 만들고,
   - `pip download` (또는 `pip wheel`)로 **타깃과 동일한 OS/Python/아키텍처** 기준 의존성 wheel을 **전이 의존까지** 수집하며,
   - **prebuilt JupyterLab 확장 wheel**도 함께 담습니다. → 폐쇄망 안에서 npm 빌드 없이 labextension 설치 가능(설치 후 새로고침으로 로드).
   - **의존성 매니페스트(SBOM류: 이름·버전·용도·라이선스·해시·전이 의존)**를 만들어 보안검토에 제출, 버전/해시는 핀 고정.
   - 폐쇄망에서는 `pip install --no-index --find-links=<번들폴더> jupyter_utils[extras]` 로 오프라인 설치.
   - 무거운 백엔드 SDK(boto3/google-cloud-storage 등)는 **extras**(`[s3]`, `[gcs]`)로 분리해 **보안검토 단위를 작게** 쪼개고 필요한 것만 반입.

## 설계 원칙

### 백엔드 비종속성 (Backend portability) — ⭐ 핵심

외부 스토리지/데이터에 접근하는 작업(예: 쿼리, 파일 읽기/쓰기)은 **특정 백엔드에 묶이지 않아야** 합니다.
사내 인프라 변경이나 이직 등으로 환경이 바뀌어도(예: AWS S3 → GCP GCS) 견고하게 쓸 수 있도록,
**추상화 계층(인터페이스) 뒤에 백엔드 구현을 숨기는** 방식을 기본으로 합니다.

- 사용자 코드는 추상 인터페이스(예: `Storage.read(path)`, `query(sql)`)만 호출합니다.
- 백엔드(S3 / GCS / 로컬FS / DB 등)는 **교체 가능한 어댑터(plugin)**로 구현합니다.
- 백엔드 선택은 **설정/환경변수**로 주입하고, 인증 정보는 코드에 하드코딩하지 않습니다.
- 새 백엔드 추가가 기존 코드 수정 없이 가능하도록(개방-폐쇄 원칙) 설계합니다.
- 폐쇄망 특성상, 특정 백엔드 SDK가 없는 환경에서도 import 단계에서 죽지 않도록 **선택적(lazy) 의존성**으로 처리합니다.
